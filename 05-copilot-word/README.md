# 05 — Copilot in Word

You've built your draft in Pages. Now it's time to turn it into a formal proposal document you can actually submit to your supervisor or management team.

---

## What Copilot Can Do in Word

- Draft new content based on a prompt
- Rewrite, shorten, or expand existing text
- Change the tone (formal, casual, persuasive)
- Summarise long documents
- Generate tables, lists, and structured sections
- Reference other files from your Microsoft 365 account

---

## Workshop Scenario: Turning Your Draft into a Proposal

Your AI Usage Guide draft is now in Word (exported from Pages). The goal here is to reframe and polish it as a **formal proposal** — something your supervisor can read, approve, and sign off on.

A proposal has a slightly different structure from a guideline. It needs to make a case for why this guide is necessary, what resources or changes are needed to implement it, and what the expected outcome is.

---

## Prompts for Copilot in Word

### Reformatting as a proposal

```
I have drafted an AI Usage Guide for my department. 
Reformat this document as a formal proposal to submit 
to management. Add the following sections if they are 
not already present:
- Executive Summary (max 150 words)
- Business Justification (why this guideline is needed)
- Implementation Steps
- Expected Outcomes
- Approval and Sign-off section

Keep the existing content but adjust the tone to be 
formal and suitable for senior management.
```

---

### Improving specific sections

```
Rewrite the Executive Summary to be more concise and 
compelling. It should clearly state the problem, the 
proposed solution, and the expected benefit in 3 sentences.
```

```
The Business Justification section needs to be stronger. 
Add 2 specific examples of risks that organisations face 
when employees use AI tools without guidelines. 
Use a professional tone.
```

```
Add a section called "Risk of Not Having This Guideline" 
that outlines 3 potential consequences for the organisation 
if AI usage remains unregulated. Keep it factual, not alarmist.
```

---

### Adjusting tone and style

```
This document is currently too casual for a management 
proposal. Rewrite it using formal business language 
throughout, while keeping it easy to read.
```

```
The Dos and Don'ts section reads like a list of rules. 
Rewrite it as a set of guidelines with brief explanations 
for each point so employees understand the reasoning 
behind each one.
```

---

### Referencing other documents

If you have other files in your SharePoint or OneDrive, you can reference them in Copilot using the `/` key:

```
Using the company IT policy document [reference the file], 
check if any of the AI guidelines in this proposal conflict 
with existing IT policies. Highlight any inconsistencies.
```

---

### Final polish

```
Review this entire document for consistency in formatting, 
tone, and language. Flag any sections that seem incomplete 
or unclear, and suggest improvements.
```

```
Create a one-page summary of this proposal that I can 
attach as a cover page. Include the title, purpose, 
key recommendations, and a signature block for approval.
```

---

## Tips for Working with Copilot in Word

- Use **Copilot in the document** (the floating Copilot icon in the margin) for inline suggestions on specific sections.
- Use **Copilot Chat in Word** (the side panel) for bigger-picture tasks like restructuring or referencing other documents.
- Always review Copilot's output before finalising. It will sometimes rephrase things in ways that don't quite match your intended meaning.
- Use **Track Changes** if you want to compare the original draft against Copilot's revisions.
