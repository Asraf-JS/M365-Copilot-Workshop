# 04 — Copilot Pages: Prompts

*Use these prompts to build, organise, and refine your AI Usage Guide draft in Copilot Pages.*

> **Session guidance:**
> Use **one main Copilot Chat session** for Part 1. This should preferably continue from your Topic 03 research session.
> After generating the draft, click **Edit in Pages** to move the content into Copilot Pages.
> Parts 2 to 14 should be done **inside the same Copilot Page**, using the Copilot panel in Pages.
> Do not start a new chat for every prompt unless the page becomes too messy or you want to test a different version. The goal is to build one continuous draft.

---

## Part 1: Rename the Chat Session

Before continuing, rename your Copilot Chat session so it is easier to find later.

Suggested chat name:

```text
AI Usage Guide Draft - [your department]
```

Use this naming pattern if you are running the activity for a specific department:

```text
AI Usage Guide Draft - HR
AI Usage Guide Draft - Finance
AI Usage Guide Draft - IT
AI Usage Guide Draft - Operations
```

> **Why this matters:**
> This topic continues from your research work in Topic 03. A clear chat name makes it easier to return to the same session when you need to refine, verify, or reuse the content.

---

## Part 2: Start Your Draft from Copilot Chat

Run this in **Copilot Chat**, then click **Edit in Pages**.

```
Based on my research, create a structured first draft of an AI Usage Guide for my department.

Include these sections:
1. Introduction and Purpose
2. Scope, including who this guide applies to
3. Approved AI Tools
4. Dos and Don'ts
5. Data Privacy Rules
6. Human Review and Accountability
7. How to Report Issues or Mistakes
8. Review and Update Schedule

Write each section with real content, not placeholder text.

Keep the language clear and suitable for non-technical staff.

Aim for 600 to 800 words total.
```

---

## Part 3: Improve the Overall Structure in Pages

Once you are in **Copilot Pages**, use this prompt in the Copilot panel.

```
Review this AI Usage Guide draft and improve the structure.

Make the document easier to read by:
- improving the section headings
- grouping related ideas together
- removing repeated points
- converting long paragraphs into shorter paragraphs or bullet points
- making the flow more logical for employees reading this for the first time

Do not remove important policy, privacy, or risk information.
```

---

## Part 4: Rewrite the Introduction

```
Rewrite the Introduction section so it starts with a short workplace scenario.

The scenario should be one sentence describing a situation where AI could help an employee complete work faster or more clearly.

After the scenario, explain the purpose of the guide.

Keep the tone professional, clear, and suitable for non-technical staff.
```

---

## Part 5: Strengthen the Dos and Don'ts Section

```
The Dos and Don'ts section is too generic.

Rewrite it with at least 3 specific examples relevant to [your department].

Use a table with two columns:
1. Do
2. Do Not

Keep each point to 1 to 2 sentences.
```

---

## Part 6: Add Department-Specific Examples

```
Add a new section called "Examples of AI Use in [your department]".

Include at least 5 practical examples.

For each example, include:
- the task
- how AI can help
- what the employee must check before using the output

Keep the examples realistic and suitable for a corporate workplace.
```

---

## Part 7: Improve the Data Privacy Section

```
The Data Privacy section needs to reference PDPA, Malaysia's Personal Data Protection Act 2010.

Add one paragraph explaining why PDPA matters when employees use AI tools.

Also include examples of information employees should avoid entering into AI tools, such as:
- personal data
- employee records
- customer information
- confidential business information
- passwords or credentials
- unpublished financial or legal information

Keep the wording practical and easy to understand.
```

---

## Part 8: Add Human Review and Accountability

```
Add a section called "Human Review and Accountability".

Explain why AI-generated content should be reviewed before it is used.

Include guidance on:
- checking facts
- verifying sources
- reviewing tone
- checking for bias
- confirming that the final decision remains with a human
- avoiding blind copy-and-paste use

Keep the tone practical and non-technical.
```

---

## Part 9: Add a Short FAQ Section

```
Add a short FAQ section at the end of the guide.

Create 5 questions an employee might ask during their first week using AI tools.

Write the answers in plain language.

Keep each answer short and practical.
```

---

## Part 10: Create a Mermaid Decision Flowchart

Use this activity to show how Copilot Pages can support visual thinking, not only text drafting.

In Copilot Pages:

1. Use the `/` menu.
2. Insert a **Code block**.
3. Select **Mermaid**.
4. Paste the Mermaid code generated by Copilot.

First, ask Copilot:

```
Create a Mermaid flowchart for employees deciding whether they should use AI for a work task.

Audience: non-technical employees.

Include these decision points:
1. Check whether confidential or personal data is involved.
2. Check whether the tool is approved.
3. Check whether manager approval is needed.
4. Use AI only for suitable tasks such as drafting, summarising, brainstorming, or analysis.
5. Review the output before using it.
6. Check facts, tone, and bias.
7. Document the use of AI when required.

Use clear Yes/No decision paths.

Give me raw Mermaid code only.

Do not include explanation.
```

---

## Part 11: Create a Mermaid Workflow for the Full Workshop Journey

Use this optional prompt if you want to show the full AI Usage Guide journey as a process diagram.

```
Create a Mermaid flowchart showing the approval workflow for our AI Usage Guide.

Include these steps:
1. Draft the guide in Copilot Pages.
2. Export the draft to Word.
3. Review and refine the document in Word.
4. Send the proposal to the manager using Outlook.
5. Manager reviews the proposal.
6. If changes are requested, revise the document.
7. If approved, create a PowerPoint briefing.
8. Share the guide with the department.
9. Collect feedback using Microsoft Forms.
10. Analyse feedback in Excel.

Use Yes/No decision paths where appropriate.

Give me the raw Mermaid code only.

Do not include explanation.
```

## Part 12: Collaborative Editing Prompt

Use this if you are editing the same Page with a colleague.

```
My colleague has added a section on enforcement and consequences.

Review the section and suggest 2 improvements to make it less punitive and more focused on guidance, responsible use, and employee support.

Present your suggestions in a table with these columns:
1. Current issue
2. Suggested improvement
3. Reason
```

---

## Part 13: Improve Readability

```
Review this entire AI Usage Guide and improve its readability.

Make the following improvements:
- simplify long sentences
- reduce repeated ideas
- make the tone consistent
- improve transitions between sections
- make the content easier for non-technical employees to understand

Do not remove important privacy, compliance, or risk information.
```

---

## Part 14: Prepare to Export to Word

Before exporting to Microsoft Word, run this prompt.

```
Review this entire document before I export it to Microsoft Word.

Flag:
- sections that repeat themselves
- tone shifts between formal and casual
- missing information that a reader would expect
- weak examples
- unclear instructions
- sections that may need manager approval

Present your review in a table with three columns:
1. Area
2. Issue
3. Suggested improvement
```

Then export the Page:

```text
Click ... > Export > Word document (.docx)
```

---

## Part 15: Final Version for Word

Use this if you want Copilot Pages to reorganise the content before export.

```
Prepare this AI Usage Guide draft to become a formal Microsoft Word proposal.

Organise it into this structure:
1. Title
2. Executive Summary
3. Background
4. Purpose
5. Scope
6. Approved AI Tools
7. Recommended Use Cases
8. Dos and Don'ts
9. Data Privacy Rules
10. Human Review and Accountability
11. Reporting Issues or Mistakes
12. Review and Update Schedule
13. Next Steps

Make the wording professional and suitable for submission to a manager.
```
