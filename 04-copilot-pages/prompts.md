# 04 — Copilot Pages: Prompts

Use these to build your AI Usage Guide draft in Copilot Pages.

---

## Starting your draft

Run this in Copilot Chat, then click **Edit in Pages**:

```
Based on my research, create a structured first draft of an AI Usage Guide for my department. Include these sections:
1. Introduction and Purpose
2. Scope (who this applies to)
3. Approved AI Tools
4. Dos and Don'ts
5. Data Privacy Rules
6. How to Report Issues or Mistakes
7. Review and Update Schedule

Write each section with real content, not placeholder text. Keep the language clear and suitable for non-technical staff. Aim for 600-800 words total.
```

---

## Refining content inside Pages

Once you're in Pages, use these prompts in the Copilot panel:

```
Rewrite the Introduction section to open with a short scenario — one sentence describing a work situation where AI could have helped. Then state the purpose of the guide.
```

```
The Dos and Don'ts section is too generic. Rewrite it with at least 3 specific examples relevant to [your department]. Keep each point to 1-2 sentences.
```

```
Add a short FAQ section at the end with 5 questions an employee might ask on their first week using AI tools. Write the answers in plain language.
```

```
The Data Privacy section needs to reference PDPA (Malaysia's Personal Data Protection Act 2010). Add one paragraph explaining why this matters for our AI use.
```

---

## Generating a Mermaid diagram

In Pages, use the `/` menu to insert a **Code block**, select **Mermaid**, then paste the output from this prompt:

```
Create a Mermaid flowchart for the decision an employee should make before using AI for a task. Include: checking for confidential data, getting approval if needed, verifying the output, and documenting what was used. Give me the raw Mermaid code only — no explanation.
```

---

## Collaborative editing prompts

If editing with a colleague in the same Page:

```
My colleague has added a section on enforcement and consequences. Review it and suggest 2 improvements to make it less punitive and more focused on guidance.
```

---

## Preparing to export

Before exporting to Word:

```
Review this entire document for consistency. Flag any sections that repeat themselves, any tone shifts between formal and casual, and any missing information that a reader would expect to find.
```

Then: click **...** > **Export** > **Word document (.docx)**
